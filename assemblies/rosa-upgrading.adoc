= Lifecycle policy and planning
[role="_abstract"]
To plan an upgrade, review the link:https://access.redhat.com/support/policy/updates/openshift/dedicated[lifecycle policies]. This information includes how many versions are supported, the CVE process, mandatory upgrade policies, and so on.

[id="rosa-upgrading_{context}"]
= Upgrading a cluster
There are three methods available to upgrade a {product-title} cluster:

* Manually through the `rosa` CLI
* Manually through the OpenShift Cluster Manager (OCM) console
* Scheduling automatic upgrades through the OpenShift Cluster Manager (OCM) console
